¿Qué es GIT? Git es un sistema de control de versiones distribuido. Es un programa que administra los cambios de un proyecto a lo largo del tiempo, permitiendo registrar cada modificación en el código y navegar entre diferentes versiones como si fuera una "máquina del tiempo".



¿Cuáles son los comandos más importantes y para qué sirven?

git init: Inicializa un proyecto para empezar a usar Git en esa carpeta.

git add: Agrega archivos al área de preparación para ser guardados.

git commit: Crea un snapshot o punto de guardado permanente con un mensaje descriptivo.

git status: Muestra el estado actual de los archivos.

git log: Muestra el historial de todos los cambios o "fotos" tomadas al proyecto.



¿Cuál es la diferencia entre Git y Github? Git es la herramienta que instalas en tu computadora para gestionar las versiones localmente. GitHub es una plataforma web (un servidor remoto) donde se suben esos proyectos para respaldarlos, compartirlos o trabajar en equipo.

Comandos para subir cambios a un repositorio: Para subir tus cambios a un repositorio remoto (como GitHub), los comandos en orden son:

git add . (para preparar todos los archivos).

git commit -m "mensaje" (para guardar el cambio localmente).

git remote add origin [URL_del_repo] (solo la primera vez, para conectar local con remoto).

git push origin master (para enviar los cambios a la nube).



¿Qué es una branch? Una branch es una versión alternativa o línea de tiempo distinta del código. Permite trabajar en nuevas funciones sin alterar la versión principal del proyecto hasta que estés seguro de que funciona
